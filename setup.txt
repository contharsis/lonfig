redshift

mkdir -p ~/.config/systemd/user
vim ~/.config/systemd/user/redshift.service
add to file
"
[Unit]
Description=Display color temperature adjuster
After=graphical.target

[Service]
ExecStart=/usr/bin/redshift -t <day>:<night> -r -l <latitude>:<longitude> -m randr
Restart=always

[Install]
WantedBy=default.target
"
day is your daily temperature, night is your nightly temperature
latitude is the latitude of your location and longitude is the longitude of your location,
they can also be floating point numbers
systemctl --user daemon-reload
systemctl --user enable redshift.service
sudo reboot

install java

go to https://www.oracle.com/java/technologies/downloads
download latest version java with Product/file description as "x64 Debian Package"
cd ~/Downloads
sudo apt-get install ./jdk-<version>_linux-x64_bin.deb
find directory where your jdk installed, mine is /usr/lib/jvm/jdk-<version>
vim ~/.bashrc, add to end of file
"
# Java
export JAVA_HOME="/usr/lib/jvm/jdk-<version>"
export PATH="$JAVA_HOME/bin:$PATH"
"

if you need to have multiple jdks
you will use the jdk installation folder "/usr/lib/jvm" to link the jdks with the environment
if your jdk has deb package, it will install automatically there when you run sudo apt-get install ./<jdk debian package>
if your jdk doesn't have deb package, download the "x64 Compressed Archive" version of the jdk
and manually put it inside "/usr/lib/jvm" for example i will download jdk 8u341

after download
cd ~/Downloads
sudo mv jdk-8u341-linux-x64.tar.gz /usr/lib/jvm
sudo tar -xzvf jdk-8u341-linux-x64.tar.gz
sudo rm jdk-8u341-linux-x64.tar.gz

create link which uses a jdk
sudo ln -s /usr/lib/jvm/<jdk>/bin/java /usr/bin/java<jdk version>
for example
sudo ln -s /usr/lib/jvm/jdk-11/bin/java /usr/bin/java11
now you can use java11 to run with java 11

everytime you want to use that jdks environment, change JAVA_HOME variable to that jdk
export JAVA_HOME="/usr/lib/jvm/<jdk>"

setup github authentication

ssh key

cd ~/.ssh
ssh-keygen -C <email>
set the file in which to save the key to <github username>_rsa
set passphrase to memorable password
eval "$(ssh-agent -s)", check if ssh-agent process is running, otherwise start it
for each ssh key / github account add this to end of file
vim ~/.ssh/config
"
Host <github username>
	User git
	Hostname github.com
	AddKeysToAgent yes
	IdentitiesOnly yes
	IdentityFile ~/.ssh/<github username>_rsa
"
go to https://github.com
Settings
SSH and GPG keys
SSH keys, click "New SSH key"
Title, set it as the purpose of the key
Key type, pick "Authentication Key"
cat ~/.ssh/<github username>_rsa.pub, copy the key
Key, paste the public key in there
click "Add SSH key"

now everytime you boot up your PC and want to use that github account
you will have to enter the passphrase for the key once, then it's stored in the ssh-agent
if you want to check the current stored fingerprints
ssh-add -l
if you want to check the current stored public key parameters
ssh-add -L

if you are transfering keys from one machine to another, you need to transfer both and set
their permission to 600 for the private and 644 for the public key
chmod 600 id_rsa
chmod 644 id_rsa.pub

git config --global credential.helper store

Developer settings
Personal access tokens

use a git command inside a repository, enter username, paste the token in password field

W: <repo url> Key is stored in legacy trusted.gpg keyring (/etc/apt/trusted.gpg), see the DEPRECATION
section in apt-key(8) for details.

sudo apt-key list
sudo apt-key export <last 8 characters of pub code together no space> | sudo gpg --dearmour -o /usr/share/keyrings/<key>.gpg
sudo vim /etc/apt/sources.list.d/<repo>.list

sudo apt update
